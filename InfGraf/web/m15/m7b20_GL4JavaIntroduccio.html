GL4Java. Introducció al GL4Java

GL4Java defineix la classe "GLCanvas" que és la classe base per a implementar aplicacions que utilitzin OpenGL des de Java. Totes les crides a OpenGL dins d'aquesta classe tenen el mateix prototipus que l'API d'OpenGL per al llenguatge C, però han d'anar prefixades pel nom de la llibreria que les implementa. És a dir, les crides a rutines de la llibreria GL van prefixades per gl. (per exemple: gl.glVertex3f) i les de rutines de la llibreria GLU van prefixades per glu. (per exemple: glu.gluLookAt)

L'únic que cal fer és crear una subclasse de la classe GLCanvas i implementar els mètodes 

          public void init ()
              /* mètode d'inicialització, es crida després que el
                 GL-Context ha estat inicialitzat. És per a inicialitzar
                 tant coses de Java com de GL */

          public void display ()
              /* mètode per a pintar, és cridat des de "paint" */

GL4Java també defineix funcions pròpies per a extendre OpenGL. Aquestes funcions d'extensió comencen totes per glj, i també van prefixades per glj. (com per a les altres llibreries). Entre elles es troben les següents: 

          protected void glj.gljInit ()
              /* Inicialitza el GL-Context. És cridada automàticament pel
                 primer "paint", per tant no cal que es cridi explícitament */

          public boolean glj.gljMakeCurrent (boolean freeContextFirst)
              /* Comprova si s'ha inicialitzat GL4Java i activa el
                 GL-Context per a aquesta execució. Aquesta crida s'ha de
                 fer en el primer mètode en que volguem usar crides a OpenGL
                 (per exemple en el nostre init(), perque ens hem
                 d'assegurar que el GL-Context estigui actiu. */

          public native void glj.gljSwap ()
              /* mètode de swap de buffers usat en el doble buffering */

Equivalències de tipus

Com que OpenGL defineix els seus propis tipus, ens cal un estil de conversió d'aquests tipus als tipus de java. El primer criteri que es té en compte és la mesura en bytes. El segon, i també convenient, és trobar tipus equivalents en java, on la conversió que s'hagi de fer sigui mínima. Tots els tipus unsigned han de convertir-se en un tipus byte de mesura equivalent de java, així doncs en aquests casos caldrà un fer un cast. Com que un programador en java utilitza taules, es genera la funció GL4Java per a tots els tipus de taules de java, per a aconseguir compatibilitat amb el GLvoid *.
