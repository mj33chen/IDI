Ombres. Càlcul per volums d'ombres. Utilització d'OpenGL

L'algorisme de simulació d'ombres usant volums d'ombra es basa en l'ús de l'stencil buffer i realitza els passos següents (en el cas senzill d'un únic focus de llum i d'un únic objecte que produeix ombra):

  • Visualitzem tota l'escena sobre el buffer de profunditat (desactivant el buffer de color).
  • Inicialitzem l'stencil buffer.
  • Visualitzem els polígons de les cares laterals del volum d'ombra, modificant l'stencil buffer a tots els pixels on pintem polígons d'ombra de manera que al final l'stencil buffer codifica les zones de pantalla on es projecten els polígons de l'escena que es troben a l'ombra.
  • Apaguem el focus de llum.
  • Visualitzem tota l'escena. Durant aquesta visualització, l'stencil buffer fa que només pintem els polígons que es visualitzen en color d'ombra.
  • Encenem el focus de llum.
  • Visualitzem tota l'escena. Durant aquesta visualització, l'stencil buffer ens impedeix que tornem a pintar els polígons que ja haviem pintat en color d'ombra.

A continuació teniu un tros de programa que genera la imatge que hem vist de l'habitació amb ombres projectades sobre les parets i els objectes. Observeu que els comentaris són enllaços a les explicacions corresponents a les diferents parts de l'algorisme.

Aquest fragment de codi s'ha extret del programa shadowvol.c, de l'Advanced OpenGL Programming tutorial del Siggraph'1999. 


void redraw_shadow(void)

{

   glClear(GL_COLOR_BUFFER_BIT|GL_DEPTH_BUFFER_BIT|GL_STENCIL_BUFFER_BIT);





       /*  Visualitzem l'escena desactivant el buffer de color  */

   glColorMask(GL_FALSE, GL_FALSE, GL_FALSE, GL_FALSE);

   render(); /*  visualitza l'escena  */





       /*  Activem l'stencil buffer i el modifiquem amb les cares frontals  */

   glEnable(GL_STENCIL_TEST);

   glDepthMask(GL_FALSE);

   glStencilFunc(GL_ALWAYS, 0, 0);

   glEnable(GL_CULL_FACE);

   glStencilOp(GL_KEEP, GL_KEEP, GL_INCR);

   glCullFace(GL_BACK);

   Dibuixa(SHADOWVOL);



       /*  Modifiquem l'stencil buffer amb les cares posteriors del volum d'ombra  */

   glStencilOp(GL_KEEP, GL_KEEP, GL_DECR);

   glCullFace(GL_FRONT);

   Dibuixa(SHADOWVOL);

   glDepthMask(GL_TRUE);

   glColorMask(GL_TRUE, GL_TRUE, GL_TRUE, GL_TRUE);

   glCullFace(GL_BACK);

   glDepthFunc(GL_LEQUAL);

   glStencilOp(GL_KEEP, GL_KEEP, GL_KEEP);

   glDisable(GL_CULL_FACE);



       /*  Apaguem la llum i visualitzem els polígons foscos  */

   glStencilFunc(GL_EQUAL, 1, 1);

   glDisable(GL_LIGHT0);

   render(); /*  visualitza l'escena  */





       /*  Encenem la llum i pintem els polígons clars de l'escena  */

   glStencilFunc(GL_EQUAL, 0, 1);

   glEnable(GL_LIGHT0);

   render(); /*  visualitza l'escena  */





   glDepthFunc(GL_LESS);

   glDisable(GL_STENCIL_TEST);



   glutSwapBuffers();

   glFlush();

}

La técnica dels volums d'ombra pot ser molt eficient si els objectes que produeixen l'ombra són simples, ja que en aquest cas és senzill calcular la silueta, les arestes del contorn i la piràmide d'ombra. El problema ens apareix quan ens trobem amb objectes de forma i silueta complexa. D'altra banda, en alguns casos patològics la forma del volum d'ombra pot produir problemes i errors numèrics en les operacions amb l'stencil buffer. En tot cas, la técnica és clarament superior a la de projecció d'ombres, ja que permet visualitzar ombres que es projecten sobre superfícies i objectes de forma complexa.
